# v0 Baseball Fan UI Reference (2026-06-05)

## 요약

- 공개로 열리는 v0 페이지 `Baseball fan app - v0 by Vercel`과 스크린샷 자산을 확인해, 응원ON MVP UI에 반영할 만한 화면 구조와 분위기를 정리했다.
- 이 참고안은 현재 앱의 MVP 제약을 유지한다. 즉, 하단 업로드 탭은 만들지 않고 업로드는 그룹 입장 후 회차 기록 안에서만 둔다.
- 원본 v0 예시는 연한 피치 배경, 흰 카드, 코랄 CTA, 네이비 팀 배지를 중심으로 한다. 이번 Android 앱 적용도 이 밝은 피치/코랄 톤을 기준으로 맞춘다.

## v0에서 확인한 핵심 화면

1. 로그인 화면
2. 회원가입 화면
3. 구단 선택 화면
4. 팀 홈 화면
5. 그룹 목록 화면
6. 응원방/회차 기록 화면

## UI 신호

- 상단은 브랜드명과 현재 화면 맥락을 분명히 보여준다.
- 팀 선택은 별도 진입 화면과 홈 상단 선택 UI 모두에서 중요한 축으로 다뤄진다.
- 팀 홈은 단순 피드만이 아니라 최근 경기 흐름, 순위, 상태 요약 같은 정보 카드를 함께 둔다.
- 그룹 화면은 카드 리스트 중심이며, 공개/비공개 여부와 멤버 수를 빠르게 읽을 수 있다.
- 응원방은 회차 단위 탐색이 핵심이며, 회차 전환과 기록 작성 영역이 분리돼 있다.
- 카드 중심 구조, 굵은 제목, 분명한 CTA, 안정적인 하단 탭 구조가 일관된다.
- 로그인 화면은 중앙 정렬 제목, 둥근 입력 필드, 코랄색 로그인 버튼을 사용한다.
- 구단 선택 화면은 큰 흰 카드, 코랄 테두리, 둥근 팀 배지, 캐러셀 느낌의 선택 흐름을 사용한다.
- 팀 홈 화면은 큰 팀 카드, `6위 / 10개 구단` 같은 순위 pill, 최근 전적 원형 칩, 직전 경기 정보 카드로 구성된다.
- 그룹 목록은 둥근 검색 입력, 흰 그룹 카드, 코랄 `들어가기` 버튼, 오른쪽 아래 코랄 FAB가 핵심이다.
- 응원방은 회차 탭, 연한 코랄 업로드 슬롯, 게시물 카드, 하단 코랄 액션 버튼을 사용한다.

## 이번 프로젝트에 적용할 번역 원칙

1. 구조는 v0를 따른다: 홈, 응원방, 그룹, 응원가, 마이의 다섯 축 유지.
2. 색감은 v0에 맞춘다: 연한 피치 배경, 흰 카드, 코랄 CTA, 네이비 배지와 텍스트 강조.
3. 그룹 생성 폼은 즉시 노출하지 않고 `응원 그룹 만들기` 버튼 뒤에 숨긴다.
4. 마이페이지에서는 상단 팀 칩을 다시 노출하지 않는다.
5. 업로드는 전역 탭이 아니라 `그룹 -> 회차 상세 -> 내 기록` 안에서만 제공한다.
