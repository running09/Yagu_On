# 프로젝트 결정 기록

## 2026-05-30 질의 결과

- 앱 이름은 우선 `Yagu_On`으로 통일한다.
- 야구 경기 및 순위 API는 별도로 사용하지 않는다.
- 경기 및 순위 정보는 웹 크롤링으로 가져오는 방향이다.
- 로그인 연동은 초기 구현 범위에서 제외한다.
- Kakao, Google 로그인 실제 SDK 연동은 일단 하지 않는다.
- 영상 업로드 길이 제한은 빠른 구현을 위해 클라이언트에서 우선 처리한다.
- 그룹 권한은 owner/admin/member 구조로 설계한다.

## 추가 설명이 필요한 항목

### 클라이언트 규칙과 서버 규칙

클라이언트 규칙은 Android 앱 안에서 사용자의 입력이나 업로드를 막는 코드다. 예를 들어 사용자가 5초를 넘는 영상을 선택하면 앱에서 바로 경고를 띄우고 업로드 버튼을 막는 방식이다.

서버 규칙은 Firebase Storage, Firestore Security Rules, Cloud Functions 같은 서버 쪽에서 데이터를 최종적으로 검사하는 규칙이다. 앱을 우회하거나 잘못된 요청이 들어와도 서버에서 한 번 더 막을 수 있다.

이 프로젝트에서는 발표 및 과제 범위에 맞춰 우선 클라이언트에서 5초 제한을 처리하고, Firebase Storage나 Cloud Functions를 실제로 붙이는 단계에서 서버 검증 범위를 다시 정하면 된다.

2026-05-30 기준 결정: 빠른 구현을 위해 5초 이하 영상 제한은 클라이언트에서 우선 처리한다. 단, 이 선택은 초기 구현을 위한 것이며, 추후 업데이트나 실제 서비스 확장 시 서버 검증을 추가할 수 있다.

### 권한 규칙

권한 규칙은 누가 어떤 데이터를 읽고, 만들고, 수정하고, 삭제할 수 있는지를 정하는 기준이다.

예시:

- 사용자는 자기 게시물만 수정할 수 있다.
- 그룹 멤버만 해당 그룹 피드를 볼 수 있다.
- 그룹에 참여한 사용자만 회차별 응원 글을 올릴 수 있다.
- 팀 정보와 경기 정보는 모든 사용자가 읽을 수 있지만, 일반 사용자는 직접 수정할 수 없다.

Firestore Security Rules는 위 권한 규칙을 Firebase DB에 적용하는 파일이다. 초기 개발 중에는 화면 기능 구현을 먼저 하고, 제출 전에는 최소한의 읽기/쓰기 제한을 정리하는 것이 필요하다.

2026-05-30 기준 결정: 그룹 권한은 owner/admin/member 구조를 사용한다. 자세한 설계는 [그룹 권한 및 회차 기록 설계](group-permissions.md)를 따른다.

## 충돌 표시

- 기존 기획 문서에는 Kakao, Google 소셜 로그인 구현이 포함되어 있었지만, 2026-05-30 기준 초기 구현 범위에서는 로그인 연동을 제외한다.
- 기존 기획 문서에는 야구 경기 및 순위 데이터 API를 추후 선정한다고 되어 있었지만, 2026-05-30 기준 별도 API 없이 웹 크롤링으로 수집하는 방향으로 변경했다.
- 기존 기획 문서와 Android 리소스에서 `응원ON`, `YAGU_ON` 표기가 섞여 있었지만, 2026-05-30 기준 `Yagu_On`으로 통일했다.
