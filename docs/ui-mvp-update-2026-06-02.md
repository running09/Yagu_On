# UI MVP 개선 요약

2026-06-02 기준으로 현재 앱 UI를 MVP 사용 흐름 중심으로 정리했다.

## 핵심 변경

- 하단 탭에는 별도 업로드 탭을 두지 않고, 업로드는 그룹 입장 후 회차 기록 안에서만 제공한다.
- 회차 기록의 미디어 영역은 터치하면 카메라 촬영, 폴더 사진 선택, 폴더 영상 선택 중 고를 수 있게 했다.
- 선택된 사진에는 미리보기를 표시하고, 입력한 글귀를 스토리처럼 미디어 위에 오버레이한다.
- 홈 피드는 빈 안내만 보이지 않도록 테스트 응원 이미지를 카드로 표시하고, 게시글 카드에서 저장소 URL을 노출하지 않게 했다.
- 홈, 그룹, 응원방, 인증 게이트, 응원가 화면의 개발자용 문구를 사용자용 문구로 낮췄다.
- 그룹 화면은 `owner/admin/UID/Firestore` 같은 내부 용어를 줄이고 응원 그룹, 멤버, 응원 회차 중심의 문구로 단순화했다.
- 응원방은 메시지 목록만 스크롤되고 입력창은 하단에 고정되도록 바꿨다.

## 현재 결정

- MVP에서는 업로드를 전역 탭이 아니라 그룹 회차 기록의 일부로 둔다.
- 업로드 글귀는 실제 미디어 파일에 합성하지 않고, 앱 화면에서 오버레이로 보여주며 회차 기록 text로 저장한다.
- 원격 게시글 미디어 썸네일 로딩은 별도 이미지 로더 도입 전까지 테스트 이미지를 대표 이미지로 사용한다.

## 남은 과제

- 실제 원격 이미지/영상 썸네일 표시를 위한 이미지 로딩 라이브러리 또는 Storage 다운로드 전략 결정.
- 회차 기록 저장 후 그룹 상세/홈 피드에서 방금 남긴 기록이 더 즉각적으로 보이도록 로딩/갱신 피드백 개선.
- 초대 UX에서 사용자 코드가 실제로 무엇인지 설명하거나 닉네임/검색 기반으로 바꾸는 설계.
